$scope.$apply()

Problem - AngularJS is unaware of update to $scope when something happens "outside it's world"

setTimeout(function () {
    // AngularJS is unaware of update to $scope
}, 2000);

$scope.$apply()

Solution to problem
setTimeout(function () {
    $scope.$apply(function () {
       //Timeout called
    });
}, 2000);

$scope.$apply()

  • $scope.$apply() automatically calls $rootScope.$digest()
  • use $timeout, $http etc. instead of vanilla JS counterparts
  • $scope.$apply() vs $scope.$apply(fn) - latter allows Angular to use try/cath in case of errors
